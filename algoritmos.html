Algoritmos de Resolução do Problema do Caixeiro Viajante

Held-Karp

Resumo

O algoritmo de Held-Karp é uma solução exata para o Problema do Caixeiro Viajante, baseado em programação dinâmica. Ele calcula a rota de menor custo que visita todas as cidades uma única vez, garantindo a solução ótima, mas com um alto custo computacional em instâncias grandes.

Funcionamento

O Held-Karp utiliza uma tabela de resultados intermediários para calcular a menor distância possível entre subconjuntos de cidades. Começando de uma cidade inicial, ele calcula a rota mínima para um subconjunto de cidades e armazena essa informação. À medida que mais cidades são incluídas, ele utiliza os resultados já calculados para otimizar o processo.

A complexidade do algoritmo é exponencial, com O(2^n * n^2), o que significa que ele se torna impraticável para problemas com muitas cidades, mas é muito eficiente para instâncias menores.

Exemplo

Este exemplo é uma simplificação muito raza, pois é inviável esplicar profundamente o funcionamento, que é mais complexo do que parece.

Suponha que temos quatro cidades: A, B, C e D. O Held-Karp calcula a rota ótima considerando todas as permutações possíveis dessas cidades. Veja o processo:

  1. Começamos com as rotas menores, como {A, B}, {A, C}, e calculamos a menor distância para esses subconjuntos.
  2. Em seguida, adicionamos mais cidades, por exemplo, {A, B, C}, e usamos os resultados anteriores para calcular a melhor rota.
  3. Continuamos até que todas as cidades sejam incluídas. A rota final será a de menor custo que visita todas as cidades e retorna à origem.

Inserção Mais Barata

Resumo

O algoritmo de Inserção Mais Barata é uma solução heurística que constrói a rota gradualmente. Ele insere as cidades uma por uma no local onde a inserção causa o menor aumento no custo total da rota.

Funcionamento

O algoritmo começa com uma rota simples entre duas cidades e vai inserindo outras cidades entre as já visitadas. Para cada cidade a ser inserida, o algoritmo testa em quais pontos da rota existente a nova cidade pode ser inserida, calculando o aumento no comprimento da rota. A cidade é então inserida no ponto que aumenta o custo o mínimo possível.

Exemplo

Considere as cidades A, B, C, D e E. Veja como o algoritmo funciona:

  1. Começamos com a rota A → B → C → A.
  2. Depois, inserimos a cidade D. Testamos todas as possíveis inserções, são elas:
    1. entre A e B: A → D → B → C → A.
    2. entre B e C: A → B → D → C → A.
    3. entre C e A: A → B → C → D → A.
  3. Escolhemos aquela que possui menor almento de custo, e continua para as outras cidades que ainda não foram visitadas

Vizinho Mais Próximo

Resumo

O algoritmo de Vizinho Mais Próximo é uma abordagem heurística que sempre escolhe a cidade mais próxima que ainda não foi visitada. Ele é simples e rápido, mas não garante a solução ótima.

Funcionamento

O algoritmo começa em uma cidade qualquer e, a cada passo, escolhe a cidade mais próxima que ainda não foi visitada. Esse processo continua até que todas as cidades tenham sido visitadas. No final, a rota é fechada retornando à cidade de origem.

Exemplo

Se temos quatro cidades A, B, C e D, o algoritmo segue os seguintes passos:

  1. Começamos na cidade A.
  2. Escolhemos a cidade mais próxima de A, digamos B.
  3. A partir de B, escolhemos a próxima cidade mais próxima, por exemplo C.
  4. Continuamos até visitar todas as cidades e, por fim, retornamos à cidade inicial A.
